# Empty States Rules

## Never empty
- Jamais une page vide sans explication
- Chaque etat vide doit avoir un message + une action

## CTA + explanation
- Expliquer pourquoi c'est vide
- Proposer l'action pour remplir ("Creer votre premier projet", "Inviter un membre")
- Le CTA doit etre le chemin le plus court vers la valeur

## Illustration
- Une illustration ou icone rend l'etat vide moins froid
- Garder le ton coherent avec le produit (fun, pro, minimal)
- Ne pas surcharger — une image simple suffit

## First-run vs no-data vs error
- **First-run** : bienvenue + onboarding, ton enthousiaste
- **No-data** : l'utilisateur a vide/filtre les donnees, ton neutre + action
- **Error** : quelque chose a echoue, ton rassurant + retry action
- Chaque type a un message et une action differents

## Actionable message
- Le message doit dire quoi faire, pas juste decrire la situation
- "Aucun projet" → mauvais
- "Creez votre premier projet pour commencer" → bon
- Le bouton d'action doit etre visible et accessible
