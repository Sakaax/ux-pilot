# Checkout Flow Rules

## Progress indicator
- Etapes clairement numerotees (1. Info → 2. Paiement → 3. Confirmation)
- L'utilisateur sait ou il en est et combien il reste

## Recap visible
- Resume de la commande toujours visible (sidebar ou top)
- Prix total, items, taxes — pas de surprise a la fin

## Trust badges
- SSL/securite pres du formulaire de paiement
- Logos des methodes de paiement acceptees
- Garantie satisfaction / politique de retour

## Guest checkout
- Permettre l'achat sans compte si possible
- "Creer un compte" optionnel apres l'achat (avec les infos deja remplies)

## Saved payment
- Proposer de sauvegarder les infos de paiement pour la prochaine fois
- Checkout en 1 clic pour les clients de retour

## Error recovery
- Erreur de paiement : message clair + option de reessayer
- Ne pas vider le formulaire apres un echec
- Proposer une methode de paiement alternative

## Form optimization
- Auto-detect du type de carte (Visa, Mastercard) via les premiers chiffres
- Formatage automatique du numero de carte (espaces tous les 4 chiffres)
- Clavier numerique pour les champs numeriques sur mobile

## Confirmation
- Page de confirmation claire avec numero de commande
- Email de confirmation envoye immediatement
- CTA vers le produit/dashboard apres l'achat

## Abandon recovery
- Si l'utilisateur quitte le checkout, email de rappel (si consenti)
- Sauvegarder le panier pour son retour
