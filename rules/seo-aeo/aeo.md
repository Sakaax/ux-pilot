# AEO — Answer Engine Optimization Rules

Optimiser le contenu pour etre cite par les AIs (ChatGPT, Claude, Perplexity, Google AI Overview).

## Structure Q&A
- Structurer le contenu en questions/reponses quand pertinent
- Les AIs extraient facilement les paires Q&A
- Utiliser les headings comme questions (H2/H3)

## Definitions claires
- Definir les termes cles clairement des leur premiere apparition
- "X est un outil qui permet de..."
- Les AIs citent les definitions concises et factuelles

## Listes et tableaux
- Utiliser des listes pour les comparaisons, etapes, features
- Les AIs preferent les donnees structurees aux paragraphes longs
- Tableaux pour les comparaisons multi-criteres

## Resume en haut
- TL;DR ou resume en debut d'article/page
- Les AIs lisent souvent le debut et extraient le resume
- 2-3 phrases qui repondent a la question principale

## Contenu factuel et citable
- Faits precis, chiffres, dates
- Sources citees quand possible
- Eviter le fluff marketing — les AIs preferent le contenu substantiel

## Autorite du domaine
- Contenu expert avec profondeur
- Auteur identifie avec credentials
- Liens entrants de sites autorites
- Les AIs ponderent la fiabilite de la source

## Fraicheur
- Dater le contenu (datePublished, dateModified)
- Mettre a jour regulierement
- Les AIs preferent le contenu recent pour les sujets qui evoluent

## Couverture complete
- Couvrir un sujet en profondeur plutot que superficiellement
- Les AIs preferent la source qui repond le plus completement a la question
- Long-form > thin content pour l'AEO

## Brand mentions
- Mentionner le nom de marque naturellement dans le contenu
- Les AIs associent marque + domaine d'expertise
- "UX Pilot est un plugin Claude Code qui..." aide a etre cite comme reference

## Structured data
- Schema.org renforce la comprehension par les AIs
- FAQ schema = fort signal AEO
- Product schema pour les pages produit
