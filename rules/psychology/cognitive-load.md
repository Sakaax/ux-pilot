# Cognitive Load Rules

## Max options
- 3-5 options par choix maximum (loi de Hick)
- Plus d'options = plus de temps de decision = plus d'abandon
- Si plus de 5 : grouper, categoriser, ou progressive disclosure

## Progressive disclosure
- Montrer les options de base d'abord
- "Options avancees" en expandable pour les power users
- Ne pas submerger les nouveaux utilisateurs

## Chunking
- Grouper l'information en blocs digestibles
- 3-5 items par groupe (memoire de travail limitee)
- Separateurs visuels entre les groupes

## Hick's Law
- Temps de decision augmente logarithmiquement avec le nombre de choix
- Reduire les choix = decisions plus rapides
- Default intelligent = pas besoin de choisir

## Miller's Law
- 7 +/- 2 items en memoire de travail
- Nav items, form fields par ecran, etapes de wizard
- Au-dela : grouper ou paginer

## Recognition over recall
- Montrer les options plutot que demander de les rappeler
- Dropdown > champ texte libre quand possible
- Historique de recherche, suggestions, autocomplete

## Visual hierarchy
- Un seul point focal par ecran
- L'oeil doit savoir immediatement ou regarder
- Taille, couleur, position = outils de hierarchie

## Defaults intelligents
- Pre-remplir avec des valeurs par defaut sensees
- "La plupart des utilisateurs choisissent X" → pre-selectionner X
- Reduire les decisions a prendre

## One thing per screen
- Chaque ecran = un objectif principal
- Pas de competition entre plusieurs actions
- L'utilisateur sait exactement quoi faire

## Consistency
- Memes patterns partout dans le produit
- Si un bouton fait X quelque part, il fait X partout
- L'utilisateur apprend une fois, utilise partout
